# Персональный AI-Хаб (Open WebUI + Ollama + Pipelines)

Этот проект представляет собой готовую сборку для запуска собственного интерфейса по работе с нейросетями. Сборка включает в себя Open WebUI (фронтенд и логика), PostgreSQL (надежная база данных), Ollama (движок для локальных моделей) и Pipelines (модуль для расширения функций и запуска Python-скриптов).

---

## Требования

Перед началом установки убедитесь, что на вашем VPS-сервере:
* Установлены **Docker** и плагин **Docker Compose** (`docker-compose-plugin`).
* Открыт порт `3000` в брандмауэре (firewall) для доступа к веб-интерфейсу.

---

## Установка и запуск

### Шаг 1. Подготовка рабочей директории
Подключитесь к вашему серверу по SSH, создайте папку для проекта и перейдите в неё:

```bash
mkdir my-ai-hub && cd my-ai-hub
```

Создайте файл `docker-compose.yaml` и вставьте в него конфигурацию манифеста.

### Шаг 2. Настройка доступов
Перед запуском необходимо заполнить пустые переменные окружения в `docker-compose.yaml`, чтобы обеспечить безопасность и подключить базу данных.

**В сервисе `db`:**
* `POSTGRES_DB: openwebui`
* `POSTGRES_USER: gpthub_admin` *(Должно совпадать с логином в блоке healthcheck)*
* `POSTGRES_PASSWORD: ваш_надежный_пароль`

**В сервисе `open-webui`:**
* `WEBUI_SECRET_KEY: случайная_строка` *(Например, сгенерируйте UUID или сложный набор символов. Используется для шифрования сессий).*
* `DATABASE_URL: postgresql://gpthub_admin:ваш_надежный_пароль@db:5432/openwebui`

*(Переменные `OPENAI_API_KEY` можно оставить пустыми, если вы планируете работать только с локальными моделями через Ollama).*

### Шаг 3. Старт контейнеров
Запустите проект в фоновом режиме командой:

```bash
docker compose up -d
```

При первом запуске Docker скачает все необходимые образы, а база данных инициализируется. Это может занять пару минут.

## Первые шаги

1. Откройте браузер и перейдите по адресу: **`http://<IP_вашего_VPS>:3000`**
2. При первом входе нажмите **Sign Up** (Зарегистрироваться) и создайте аккаунт. 
   > **Важно:** Первый зарегистрированный пользователь автоматически получает права Администратора.
3. В целях безопасности сразу после входа рекомендуется зайти в *Settings (Настройки)* -> *Admin Settings (Настройки администратора)* и отключить тумблер **Enable New Sign Ups** (Разрешить новые регистрации), чтобы посторонние не могли создать аккаунт на вашем сервере.

---

## Скачивание нейросетей (Ollama)

Изначально Ollama поставляется пустой. Чтобы начать общаться с ИИ:
1. В интерфейсе Open WebUI перейдите в **Settings -> Admin Settings -> Models**.
2. В разделе *Pull a model from Ollama.com* введите название нужной модели (например, `llama3.1`, `gemma2` или `qwen2.5`).
3. Нажмите кнопку скачивания. Дождитесь окончания загрузки (зависит от скорости интернета на вашем VPS).
4. Вернитесь на главный экран — скачанная модель появится в выпадающем меню сверху.

## Структура хранения данных

Конфигурация настроена так, чтобы ваши данные не потерялись при обновлении или перезапуске контейнеров. В папке с проектом автоматически создадутся директории:
* `postgres_data/` — файлы базы данных (сохраняет историю чатов, настройки и пользователей).
* `ollama_data/` — веса скачанных нейросетей (чтобы не качать их заново).
* `pipelines_logic/` — директория для ваших Python-скриптов и кастомной логики модуля Pipelines.</IP_вашего_VPS>
